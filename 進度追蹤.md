## 問題清單 (按修復順序)

### 🔴 CRITICAL

#### C3 — cache_flush_all 暫存緩衝未 sector-aligned ✅ 已修復

| 項目 | 內容 |
|------|------|
| 檔案 | `ram_cache.c:88` |
| 問題 | `malloc(total_len)` 不保證 512-byte 對齊，NO_BUFFERING WriteFile 要求 buffer 也必須 sector-aligned |
| 影響 | flush 可能失敗造成資料遺失 |
| 修法 | 改用 `VirtualAlloc` pre-allocated flush buffer (16MB) 於 RAM_CACHE struct，保證 page-aligned |
| 狀態 | ✅ 已完成，編譯成功 |

#### C1 — highest_byte_written 非原子更新 ✅ 已修復

| 項目 | 內容 |
|------|------|
| 檔案 | `fuse_bridge.c` |
| 問題 | 多執行緒並發寫入時 read-modify-write 不原子，可造成 highest_byte_written 倒退 |
| 影響 | statfs 回報容量計算錯誤 |
| 修法 | 改用 `InterlockedCompareExchange64` 原子 CAS while-loop 實現 atomic max |
| 狀態 | ✅ 已完成，編譯成功 |

#### C2 — g_open_files[] 無記憶體屏障 ✅ 已修復

| 項目 | 內容 |
|------|------|
| 檔案 | `fuse_bridge.c:58-62` |
| 問題 | 先寫入 array 再 `count++`，無 barrier 保護 |
| 影響 | 另一 thread 看到 count++ 後讀到未初始化 array entry |
| 修法 | 改用 `InterlockedIncrement` 取代普通 `g_open_count++` |
| 狀態 | ✅ 已完成，編譯成功 |

#### C4 — 每批次 malloc 高壓力 OOM ✅ 已修復

| 項目 | 內容 |
|------|------|
| 檔案 | `ram_cache.c:88-94` |
| 問題 | 每 batch 都 malloc/free 最大 16MB，高壓下堆積碎片化 |
| 影響 | malloc 失敗略過 batch，最終累積大量髒頁無法 flush |
| 修法 | 改用 cache_init 時一次性 `VirtualAlloc` 預先配置 16MB flush buffer（與 C3 合併修復） |
| 狀態 | ✅ 已完成，編譯成功 |

#### C5 — FUSE thread 無終止同步 ✅ 已修復

| 項目 | 內容 |
|------|------|
| 檔案 | `fuse_bridge.c:390-397` |
| 問題 | thread handle 立即 CloseHandle，無法等待終止 |
| 影響 | unmount 後 callback 仍在跑時 cleanup 可能競爭 |
| 修法 | 保留 thread handle 至 `g_fuse_thread_handle`，`fuse_unmount_volume` 時 WaitForSingleObject + CloseHandle |
| 狀態 | ✅ 已完成，編譯成功 |

### 🟠 HIGH

#### H1 — raid_unlink/raid_release 雙重查找 ✅ 已修復

| 項目 | 內容 |
|------|------|
| 檔案 | `fuse_bridge.c:67-75` (原) |
| 問題 | 先 find 再 remove（內部又 find），浪費 double search |
| 修法 | 將 `remove_open_file` 改為 `remove_open_file_get_size`，一次操作回傳檔案大小，raid_unlink/raid_release 不再雙重查找 |
| 狀態 | ✅ 已完成，編譯成功 |

#### H2 — statfs 空間計算未考慮 fragmentation ✅ 已修復

| 項目 | 內容 |
|------|------|
| 檔案 | `fuse_bridge.c` |
| 問題 | `used = highest_byte_written - freed` 線性累計不反映真實 |
| 影響 | 檔案 truncate 後反覆寫入會報錯容量 |
| 修法 | 新增 `total_bytes_consumed` atomic counter；statfs 改為 `consumed - freed` 公式 + cap |
| 狀態 | ✅ 已完成，Pipeline 測試 PASS |

#### L5 — realloc 直接更新指標，失敗時舊指標遺失 ✅ 已修復

| 項目 | 內容 |
|------|------|
| 檔案 | `disk_scanner.c:43` |
| 問題 | `disks = (DISK_INFO*)realloc(...)` 若 realloc 失敗直接覆蓋指標，舊記憶體遺失 |
| 修法 | 先存入 temp variable，確認成功後才賦值回 `disks` |
| 狀態 | ✅ 已完成，編譯成功 |

#### H3 — Superblock 讀取無版本比較 ✅ 已修復

| 項目 | 內容 |
|------|------|
| 檔案 | `superblock.c:128-137` |
| 問題 | 找到第一個 valid 就 break，不比較 CRC/version |
| 影響 | 多碟 superblock 不一致時載入舊版 |
| 修法 | 掃描全部磁碟收集所有 valid superblock；以 `virtual_total_bytes` + `disk_count` 為基準選最佳版本 |
| 狀態 | ✅ 已完成，IO Verify: PASS 驗證過 |

#### H4 — FUSE -s 與 fuse_loop_mt 衝突 ✅ 已修復

| 項目 | 內容 |
|------|------|
| 檔案 | `fuse_bridge.c:367` (原) |
| 問題 | mount args 設 `-s` 但 thread 用 `fuse_loop_mt` |
| 影響 | 失去多執行緒並行處理的好處 |
| 修法 | 移除 FUSE mount args 中的 `-s`，保留 `fuse_loop_mt` |
| 狀態 | ✅ 已完成，編譯成功 |
