# RAIDTEST 專題需求與實作報告（確認用）

## 1. 專題目標

本專題的目標，是在 **Windows 環境** 下，將 2～4 顆（未來可擴充更多）不同容量、不同品牌、不同介面類型的硬碟，透過軟體方式整合成一個可管理的邏輯儲存空間，並提供一個簡單、直觀、可操作的 GUI 介面，讓使用者可以完成以下工作：

1. 掃描本機可用硬碟。
2. 依實際環境動態選擇可加入 RAID 的磁碟。
3. 建立 RAID0 / RAID1 / RAID10（目前核心以 RAID0 為主，RAID1/10 作為擴充或規劃）
4. 自動寫入與讀取 metadata，讓重開機後仍能恢復陣列。
5. 提供掛載、卸載、載入、摧毀、恢復等完整流程。
6. 提供 Planner（容量規劃）與 Benchmark（效能測試）功能。
7. 記錄事件與錯誤，方便除錯與展示。
8. 讓專題具備可展示、可驗證、可交付的完成度。

這個專題的核心不是做成企業級商用 RAID，而是做成一個 **架構清楚、功能完整、可展示、可驗證** 的 Windows RAID 原型系統。

---

## 2. 專案定位

本專案的定位如下：

* **平台**：Windows
* **語言**：C / C++
* **UI**：Dear ImGui + Win32 + DirectX 11
* **儲存層**：使用者模式 RAID / Volume 管理原型
* **核心理念**：

  * 將不同容量硬碟整合成一個邏輯卷
  * 盡量提升容量利用率
  * 提供可用的管理介面
  * 支援 metadata 與 restore
  * 提供測試、壓力測試與文件化

## 3. 目前應該具備的功能清單

以下是這套軟體「應該具備」的功能，這也是你拿來確認 OPENCODE 是否做完的基準。

### 3.1 硬碟偵測與管理

#### 必要功能

* 掃描本機實際可用磁碟
* 顯示每顆磁碟資訊
* 依實際硬碟數量動態顯示，不得寫死 disk0~disk3
* 顯示磁碟：

  * 型號（Model）
  * 序號（Serial Number）
  * 容量（Capacity）
  * 匯流排類型（Bus Type）
  * SSD / HDD
  * 是否在線（Online / Offline）
  * 是否已加入 RAID
  * 裝置路徑

#### 實作方式

* 使用 Windows API 或封裝層讀取硬碟資訊
* 建立 `Device Manager` 統一處理磁碟資料
* UI 不直接掃描硬碟，只透過 Device Manager 取得資料
* 每次開啟程式或按 Refresh 時，重新更新磁碟列表

#### 目的

* 避免磁碟數量與 UI 顯示不一致
* 避免硬碟順序改變造成誤選
* 提供真實環境下的管理能力

---

### 3.2 RAID 建立與管理

#### 必要功能

* 建立 RAID volume
* 支援至少 RAID0
* 依使用者選取的磁碟建立陣列
* 正確處理建立失敗時的 rollback
* 能夠掛載與卸載
* 能夠載入既有 volume
* 能夠摧毀 volume

#### 實作方式

* 建立 `Volume Manager`
* `cmd_*` 不直接做 RAID 邏輯，只呼叫 `raid_service`
* `raid_service` 再轉給 `Volume Manager`
* Volume Manager 負責：

  * create
  * load
  * mount
  * unmount
  * destroy
  * expand
  * mirror
  * rebuild
* 每一個操作都要有明確回傳值與錯誤碼

#### 目的

* 將 RAID 工作流程模組化
* 避免 CLI、GUI 直接碰底層邏輯
* 方便後續擴充 RAID 模式

---

### 3.3 Metadata 與 Restore

#### 必要功能

* 產生與保存 volume UUID
* 保存每顆磁碟的序號與角色資訊
* 支援 superblock 版本管理
* 支援 metadata 驗證
* 支援重開機後的自動恢復
* 依序號比對，不得只靠 drive letter 或掃描順序

#### 實作方式

* 建立 `Metadata Manager`
* Metadata 至少應包含：

  * Volume UUID
  * Disk UUID / Disk Serial
  * RAID level
  * Stripe size
  * Disk count
  * Version
  * Checksum
  * Created time
  * Last mount time
* 載入時先驗證 metadata
* 若 metadata 損毀，應拒絕掛載並回報錯誤
* 若磁碟順序變動，仍可透過 serial / UUID 正確配對

#### 目的

* 避免硬碟位置改變後誤配
* 確保陣列可恢復
* 提升資料一致性與穩定性

---

### 3.4 Planner（容量規劃）

#### 必要功能

* 顯示選定磁碟的總容量
* 顯示 RAID0 可用容量
* 顯示效率與浪費空間
* 顯示不同 RAID 模式的容量差異
* 對不同硬碟組合給出即時預估

#### 實作方式

* 建立 `Planner Engine`
* Planner 與 CLI 分離，GUI 透過 service 呼叫
* 根據選定磁碟集合計算：

  * 可用容量
  * 利用率
  * 理論效率
  * 未使用容量
* RAID0：總容量接近所有磁碟總和
* Mirror：容量以最小磁碟為基準
* RAID10：容量與鏡像數量有關

#### 目的

* 讓使用者清楚知道不同硬碟怎麼組最划算
* 這是專題的重要展示亮點

## 4. 應該採用的架構

建議架構如下：

```text
GUI / CLI
   ↓
Service Layer (raid_service)
   ↓
Managers
   ├─ Device Manager
   ├─ Volume Manager
   ├─ Metadata Manager
   ├─ Planner Engine
   ├─ Event Bus
   ├─ Logger
   ├─ UI Model
   └─ Cache / Journal
   ↓
Storage Engine
   ↓
Windows / WinFsp / File System Layer
   ↓
Physical Disks
```

### 設計原則

* GUI 與 CLI 不得直接碰底層 RAID 演算法
* 任何磁碟資訊都由 Device Manager 提供
* 任何 metadata 操作都由 Metadata Manager 處理
* 任何容量計算都由 Planner Engine 處理
* 任何 RAID 狀態變更都透過 Service Layer

## 5. 建立 RAID 的實作流程

以下是標準流程，應該符合目前專案的工作方式：

1. **Scan**

   * 掃描可用磁碟
   * 顯示磁碟列表

2. **Select Disk**

   * 使用者勾選要加入的磁碟
   * 檢查磁碟是否可用、序號是否有效

3. **Planner**

   * 根據選定磁碟計算容量
   * 顯示 RAID0 / RAID1 / RAID10 的預估容量

4. **Validate**

   * 驗證容量、數量、磁碟狀態
   * 驗證 metadata 與 UUID

5. **Create**

   * 寫入 metadata
   * 建立 stripe / mirror / cache 結構

6. **Mount**

   * 以 WinFsp 或既有掛載機制掛載 volume
   * 更新狀態為 mounted

7. **Operate**

   * 讀寫檔案
   * 更新 log 與 event

8. **Unmount / Load / Restore**

   * 卸載後可重新載入
   * 依 metadata 與 serial 恢復

9. **Destroy**

   * 清除 pool / metadata / journal / config
   * 返回初始狀態

## 7. 錯誤處理原則

這套軟體必須遵守以下原則：

1. 任何 I/O 都要檢查回傳值
2. 任何 Windows Handle 都要正確關閉
3. 任何 thread 都要能正常結束
4. 任何建立失敗都要 rollback
5. 任何 metadata 不一致都要拒絕掛載
6. 任何非法 state 都不能執行命令
7. 任何 bug 都要有對應 regression test

---

## 8. 測試標準

### 必測項目

* 只有 3 顆硬碟時，UI 是否只顯示 3 顆
* 把磁碟順序換掉後能不能 restore
* 改變 drive letter 後能不能 restore
* metadata 損毀能不能拒絕掛載
* serial mismatch 時是否拒絕錯誤磁碟
* 少一顆磁碟是否進入 degraded 或失敗狀態
* destroy 後是否清理乾淨
* benchmark 是否能輸出結果
* GUI 按鈕是否會依 state 正確變化

### 壓力測試

* 大量讀寫
* 隨機 I/O
* 多 thread 同時操作
* 中斷後恢復
* 長時間運行
