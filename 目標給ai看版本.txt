# 一、專案目標（Project Goal）

本專案是一個 Windows 平台上的 RAID0 驗證專案。

目的不是打造企業級 RAID 軟體。

也不是取代 Windows Storage Spaces。

更不是做 NAS。

更不是做 Linux mdadm。

本專案主要目標只有一個：

> **驗證不同容量、不同速度的 SSD / HDD 是否可以透過軟體整合成一個大型 Volume，並提供簡單 GUI 操作與完整實驗驗證。**

本專案定位：

> **大學畢業專題**

因此：

完成度

>

可靠度

>

程式架構

>

Demo 展示

遠高於

功能數量。

---

# 二、專案核心理念

核心理念只有四句：

1.

不同容量可以整合

例如

256GB

500GB

1TB

2TB

全部組成一個 Volume。

不是只能同容量。

---

2.

不同速度可以共同使用

例如：

NVMe

SATA SSD

HDD

全部共同組成 RAID0。

容量最大化。

---

3.

不要浪費容量

不要因 RAID 傳統限制造成：

500G + 2TB

最後只能得到 1TB。

而是盡可能利用全部容量。

（Planner 會協助規劃）

---

4.

Windows 一般使用者可以操作

GUI 為主。

CLI 為輔。

## 我建議再加一份更重要的文件：**AI_GUARDRAILS.md**

如果 `AI_PROJECT_SPEC.md` 是「專案說明書」，那 `AI_GUARDRAILS.md` 就是「AI 行為規範」。

裡面會明確規定，例如：

* **不能因為覺得功能不足就自行新增 RAID5、Linux、Driver 等大型功能。**
* **不能重構已穩定的架構，只能在現有架構上改進。**
* **所有修改都要以「大學專題完成度」為第一優先，而不是企業級產品。**
* **每次開始工作前，都要先輸出目前完成度、此次修改目的、可能影響，再開始修改。**
* **每次完成後，都要產出固定格式的 Sprint Report、變更摘要、風險分析、下一步建議。**

這兩份文件一起使用，幾乎所有 AI（包括 OPENCODE）都能在新的對話中快速理解你的專案定位，不容易偏離方向，也能維持一致的開發流程。這會比只靠聊天上下文穩定得多。
